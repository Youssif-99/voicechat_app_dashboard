# لوحة تحكم تطبيق غرف الدردشة الصوتية

لوحة إدارة (Admin Dashboard) مبنية بـ **Next.js 14 + TypeScript + Tailwind CSS + Prisma (SQLite)**، مربوطة بقاعدة بيانات حقيقية وليست مجرد تصميم شكلي.

## المزايا المتضمنة

- **تسجيل دخول آمن** للأدمن (كلمة مرور مشفّرة + جلسة موقّعة عبر JWT في كوكيز httpOnly).
- **طلبات الوكالات**: عرض، قبول/رفض، إيقاف/إعادة تفعيل، تحديد المستوى (عادي/فضي/ذهبي/ماسي) ونسبة العمولة.
- **المدفوعات والأرباح**: سجل كل المعاملات (شراء عملات، اشتراك VIP، هدايا، عمولات وكلاء، شراء إطارات ودخوليات) مع إحصائيات وتصفية حسب النوع.
- **إدارة المستخدمين**: بحث، حظر (يوم واحد / 3 أيام / دائم / حظر شبكة على مستوى الجهاز)، رفع الحظر، تعديل الاسم والصورة.
- **إدارة الغرف**: تعديل اسم وصورة الغرفة، حظر/رفع حظر، وعرض حد المقاعد (20 مقعد).
- **صلاحيات سوبر أدمن**: صفحة خاصة لإضافة/حذف حسابات المشرفين، لا يراها إلا السوبر أدمن.

هذا المشروع **سكيلتون (scaffold) حقيقي وجاهز للتشغيل والتوسعة**، وليس مجرد mockup — كل شاشة تقرأ وتكتب من قاعدة بيانات فعلية. يمكنك لاحقًا استبدال SQLite بـ PostgreSQL أو MySQL بسهولة (سطر واحد في `.env` + `prisma/schema.prisma`)، وربط بوابات دفع حقيقية (مثل Stripe أو أي بوابة محلية) داخل `src/app/(dashboard)/payments`.

## التشغيل محليًا

يتطلب [Node.js](https://nodejs.org) نسخة 18 أو أحدث.

```bash
# 1. تثبيت المكتبات
npm install

# 2. إنشاء ملف البيئة (تم إنشاؤه مسبقًا هنا بقيم افتراضية تصلح للتجربة)
cp .env.example .env

# 3. إنشاء قاعدة البيانات وتوليد الجداول
npm run db:push

# 4. تعبئة بيانات تجريبية (وكالات، مستخدمين، غرف، معاملات) + حساب السوبر أدمن الأول
npm run db:seed

# 5. تشغيل السيرفر محليًا
npm run dev
```

افتح المتصفح على `http://localhost:3000` وسجل الدخول بـ:

```
admin@example.com
admin123
```

**مهم:** غيّر كلمة المرور هذه فورًا بعد أول تشغيل، وأنشئ حساب سوبر أدمن جديد من صفحة "المشرفون"، ثم احذف هذا الحساب التجريبي.

## هيكل المشروع

```
prisma/schema.prisma       نموذج قاعدة البيانات (المشرفون، الوكالات، المستخدمون، الغرف، المدفوعات)
prisma/seed.ts              بيانات تجريبية أولية
src/lib/auth.ts              تسجيل الدخول، تشفير كلمات المرور، إدارة الجلسات
src/middleware.ts            حماية صفحات اللوحة من الوصول غير المصرح به
src/app/login                صفحة تسجيل الدخول
src/app/(dashboard)          كل صفحات اللوحة (المسار محمي تلقائيًا)
  ├─ dashboard                نظرة عامة وإحصائيات
  ├─ agencies                 طلبات الوكالات
  ├─ payments                 المدفوعات والأرباح
  ├─ users                    إدارة المستخدمين والحظر
  ├─ rooms                    إدارة الغرف
  └─ admins                   إدارة حسابات المشرفين (سوبر أدمن فقط)
src/components                عناصر واجهة مشتركة
```

## الخطوات التالية المقترحة

- ربط تسجيل الدخول والتسجيل الفعلي من تطبيق الجوال (iOS/Android) بنفس قاعدة البيانات عبر REST API إضافي في `src/app/api`.
- ربط بوابة دفع حقيقية (Stripe، أو بوابات محلية مثل PayMob/Fawry حسب السوق المستهدف) لتنفيذ عمليات شراء العملات والاشتراكات فعليًا بدل الإدخال اليدوي.
- إضافة سجل تدقيق (Audit Log) لكل إجراء يقوم به الأدمن (من حظر أو تعديل)، مفيد جدًا عند تعدد المشرفين.
- ترحيل قاعدة البيانات من SQLite إلى PostgreSQL قبل الإطلاق الفعلي لدعم عدد أكبر من المستخدمين.
